Ruff is pointed at `Dev/pyproject.toml` via `--config` for reliable discovery from repo-root invocation. Hook scope is repo-wide (PRC-004).
- ADR 0001 (`docs/adr/0001-contract-ir.md`) documenting the single-Pydantic-IR, multiple-frontends architecture with Mermaid + ASCII diagrams, non-goals (no runtime codegen, no dynamic schema extension, opaque version semantics), and explicit empty-contract (scaffold / observe-only) semantics; ADR index at `docs/adr/README.md`; linked from `README.md` and `CONTRIBUTING.md` (PRC-005).

- `precept.contract.schema` with `HandoffContract` and `ContractFields` Pydantic v2 models; `extra="forbid"`, `name` regex (`^[a-z][a-z0-9_-]*$`), `mode: Literal["block", "warn"]`, bounded `min_fidelity` required when `preserved_entities` is non-empty, `required_fields`/`forbidden_drops` overlap rejection, list-duplicate rejection; `HandoffContract.__init__` wraps `pydantic.ValidationError` into `ContractValidationError`. Empty contracts (all three rule lists empty) are permitted as a deliberate scaffold mode per ADR 0001 (PRC-006).
- Shared Hypothesis strategies at `Dev/tests/unit/contract/strategies.py` (`valid_contract_names`, `valid_min_fidelity`, `valid_field_names`) reusable by PRC-008 and PRC-012 (PRC-006).
- Unit tests at `Dev/tests/unit/contract/test_schema.py` covering 38 cases including the required-when-preserved, overlap, duplicate, and property-based paths (PRC-006).
- `precept.contract.yaml_loader` with `load_contract(str | Path) -> HandoffContract` and `load_contract_from_string(str) -> HandoffContract`; uses `yaml.safe_load` exclusively, surfaces `(line, column)` on syntax errors via `ContractValidationIssue.yaml_mark`, rejects empty / non-mapping / unknown-key YAML with actionable messages; supports YAML anchors and aliases natively (PRC-007).
- Example contract YAML files at repo root: `examples/contracts/researcher_to_summariser.yaml` (blocking, fidelity 0.75) and `examples/contracts/summariser_to_writer.yaml` (warn, fidelity 0.70, mode defaulted) (PRC-007).
- Unit and integration tests for the YAML loader (`tests/unit/contract/test_yaml_loader.py`, `tests/integration/test_yaml_loader_integration.py`) covering example-file round-trip, anchor/alias resolution, malformed YAML with line/column, semantic round-trip, and contract distinctness (PRC-007).
- Runtime dependencies `pydantic>=2.5,<3` and `pyyaml>=6.0,<7` in `Dev/pyproject.toml`; dev additions `pip-tools>=7.4,<8` and `types-PyYAML>=6.0,<7` for lockfile generation and strict-mode type checking (PRC-006, PRC-007).
- `Dev/requirements-dev.lock` generated via `pip-compile --extra dev` for reproducible dev environments; regenerate on any dependency change per CLAUDE.md.
- Three follow-up tickets in `ISSUES.md`: PRC-006a (JSON Schema export helper), PRC-007a (YAML loader error enrichment with line/column on schema violations), PRC-007b (`examples/contracts/README.md`); all P2, post-MVP.
- `precept.scoring.base` module with the four anchor types of the scoring layer: `HandoffPayload` (Pydantic; `fields: dict[str, Any]`, optional `raw`, string-keyed `metadata`, `extra="forbid"`), `FieldScore` (per-field result with bounded `score in [0,1]`, `method`, `passed`), `ScoreResult` (aggregate `overall_score in [0,1]`, `field_scores`, `scorer_name`, `scorer_version`, `timestamp_iso`), and the `Scorer` ABC with the abstract `score(source, target, contract) -> ScoreResult` method (PRC-010).
- `Scorer.__init_subclass__` enforcement: every concrete subclass must declare non-empty class-level `name: str` and `version: str` attributes; failure raises `TypeError` at class-definition time. Intermediate abstract subclasses (those still carrying unimplemented abstract methods) are exempted via `inspect.isabstract`, so layered hierarchies remain possible (PRC-010).
- `Scorer` docstring codifies the implementation contract for `score()` (determinism, non-mutation of inputs, tolerance for missing optional fields, thread-safety) and reaffirms the constructor-time-load discipline. The interface is deliberately method-agnostic: no mutual-information vocabulary in the ABC, so the Phase 2 `CalibratedScorer` (PRC-035) drops in without an interface break (PRC-010).
- `ScoreResult.timestamp_iso` field validator enforcing UTC-offset ISO 8601 strings: rejects naive datetimes and non-UTC offsets, normalises a trailing `Z` suffix to `+00:00` so behaviour is consistent across the `>=3.10` Python floor (Python 3.10's `datetime.fromisoformat` does not accept `Z`; 3.11+ does). Locks the timestamp convention down before PRC-013's `ViolationEvent` introduces a sibling field (PRC-010).
- Unit tests at `Dev/tests/unit/scoring/test_base.py` (35 cases) covering ABC instantiation guard, `__init_subclass__` enforcement (missing `name`, missing `version`, empty string, non-string), intermediate-abstract exemption, positive subclass round-trip, `HandoffPayload` value polymorphism + extra-field rejection, `FieldScore` and `ScoreResult` range validation including boundary values, `timestamp_iso` UTC-offset validation (`+00:00` accepted, `Z` normalised, naive rejected with explicit message, non-UTC offset (`+01:00`) rejected, garbage rejected), and module `__all__` sanity (PRC-010).
- `precept.scoring.embedding_proxy.EmbeddingProxy`: v0 concrete `Scorer`. Sentence-transformers cosine-similarity proxy on `all-MiniLM-L6-v2` by default (`paraphrase-MiniLM-L3-v2` and arbitrary HF model identifiers swappable via `model_name`). Per-field cosine on `contract.fields.required_fields`; unweighted-mean `overall_score`; missing target field scores `0.0` with `passed=False`; non-string field values stringified via `str()`; `method="embedding_cosine"` on every `FieldScore`; clamps cosine to `[0,1]` to absorb floating-point and adversarial-pair edges (PRC-011).
- Constructor-time model load with INFO log of the resident-memory footprint constant (no `psutil` dependency); `torch.set_grad_enabled(False)` set at construction for inference-only operation (PRC-011).
- `_skip_model_load: bool = False` test-only escape hatch on `EmbeddingProxy.__init__`: skips model load and causes `score()` to raise `RuntimeError` with explicit guidance. Used by unit tests to exercise scorer wiring without the ~80 MB `all-MiniLM-L6-v2` download. Documented as private; not part of the public API (PRC-011).
- Empty-`required_fields` contracts (ADR 0001 scaffold/observe-only mode) score `overall_score=1.0` with `field_scores=[]`. Vacuous pass aligns with PRC-013's `all(field_score.passed)` aggregation (PRC-011).
- Scorer-level `min_fidelity` fallback: when `contract.fields.min_fidelity` is `None` and `required_fields` is non-empty, `EmbeddingProxy` applies a `0.5` cosine threshold and logs a `WARNING` once per `score()` call. The default lives in the scorer, not in `ContractFields`, to keep the IR generic for the Phase 2 `CalibratedScorer` (PRC-035): a calibrated MI scorer would pick its own threshold from a calibrated probability distribution, where `0.5` cosine has no meaning. See `embedding_proxy` module docstring (PRC-011).
- Unit tests at `Dev/tests/unit/scoring/test_embedding_proxy.py` (18 cases): identical-near-perfect (≥0.99), unrelated-low (<0.3), missing-target zero, field-count match, timestamp UTC-offset round-trip, empty-required vacuous pass with no warning, `min_fidelity=None` warning + 0.5 fallback (passing and failing branches), non-string and list-of-string stringification, determinism over 10 runs, `paraphrase-MiniLM-L3-v2` model-swap, `_skip_model_load` `RuntimeError`, async-via-`asyncio.to_thread` non-blocking smoke (sync API safely dispatchable from coroutines; native-async deferred to Phase 2). Integration test at `Dev/tests/integration/scoring/test_embedding_proxy_integration.py` exercising a degraded handoff against a 0.75-fidelity contract (PRC-011).
- Runtime dependencies `sentence-transformers>=2.2,<4` and `numpy>=1.24,<3` in `Dev/pyproject.toml`; narrow `[[tool.mypy.overrides]] ignore_missing_imports = true` for `sentence_transformers.*` and `torch.*` (no global escape hatch). `Dev/requirements-dev.lock` regenerated via `pip-compile pyproject.toml --extra dev` (PRC-011).
- `precept.evaluator` package scaffold (`Dev/src/precept/evaluator/__init__.py`) with empty `__all__`. Submodule names import directly until PRC-026 publishes the public surface (PRC-012).
- `precept.evaluator.rules` module exposing the three v0 rule evaluators as pure module-level functions: `required_fields_rule`, `preserved_entities_rule`, `forbidden_drops_rule`. Each is stateless, side-effect-free, and never raises - rule failures are returned as `RuleResult` data so the PRC-013 `Evaluator` aggregates them without exception-flow plumbing (PRC-012).
- `RuleResult` Pydantic model: `rule_name: str`, `passed: bool`, `details: dict[str, Any]` (default empty), `violation_message: str | None`. `ConfigDict(extra="forbid")`. `violation_message` is `None` on pass and human-readable on fail; per-rule `details` shape is documented in each rule's docstring and treated as a public contract for PRC-013's `ViolationEvent` renderer (PRC-012).
- `required_fields_rule(source, target, required)`: presence check on `target.fields`. `details = {"required": list[str], "missing": list[str]}` preserving the `required` order. `source` is part of the uniform rule signature but unused; documented in the docstring (PRC-012).
- `preserved_entities_rule(source, target, entities)`: case-insensitive substring matching on the newline-joined stringified values of `HandoffPayload.fields` (NOT `.raw`; `.raw` is optional and would couple rule behaviour to producer-population habits). Vacuous-pass when an entity is absent from source - mirrors `forbidden_drops_rule` semantics. NER-based matching deferred to PRC-030 (PRC-012).
- `forbidden_drops_rule(source, target, forbidden)`: structural drop check; flags fields present in `source.fields` but absent in `target.fields`. Target-only fields pass. `details = {"forbidden": list[str], "dropped": list[str]}` (PRC-012).
- Unit tests at `Dev/tests/unit/evaluator/test_rules.py` (29 cases): per-rule positive/negative/empty-list paths, single- and multi-missing message templates (singular vs plural form, including the verbatim AC exemplar `"Required field 'hypothesis' missing in target payload"`), case-insensitive entity matching in both source-upper/target-lower and source-lower/target-upper directions, vacuous-pass for entities absent from source, non-string field stringification, `.raw`-not-consulted assertion, no cross-field-boundary substring collisions, target-only and absent-from-both branches for forbidden drops, `RuleResult` `extra="forbid"` rejection. Hypothesis property tests: rules never raise on arbitrary input (random unicode keys, mixed-type values), and `required_fields_rule(any, target_with_those_fields)` always passes (PRC-012).
- `[tool.pytest.ini_options] pythonpath = ["tests/unit/contract"]` in `Dev/pyproject.toml` so the shared `valid_field_names` Hypothesis strategy in `tests/unit/contract/strategies.py` (originally placed there with PRC-012 named in its docstring as a reuse target) is importable from any test module without sys.path manipulation in conftest (PRC-012).
