- `precept.scoring.base` module with the four anchor types of the scoring layer: `HandoffPayload` (Pydantic; `fields: dict[str, Any]`, optional `raw`, string-keyed `metadata`, `extra="forbid"`), `FieldScore` (per-field result with bounded `score in [0,1]`, `method`, `passed`), `ScoreResult` (aggregate `overall_score in [0,1]`, `field_scores`, `scorer_name`, `scorer_version`, `timestamp_iso`), and the `Scorer` ABC with the abstract `score(source, target, contract) -> ScoreResult` method (PRC-010).
- `Scorer.__init_subclass__` enforcement: every concrete subclass must declare non-empty class-level `name: str` and `version: str` attributes; failure raises `TypeError` at class-definition time. Intermediate abstract subclasses (those still carrying unimplemented abstract methods) are exempted via `inspect.isabstract`, so layered hierarchies remain possible (PRC-010).
- `Scorer` docstring codifies the implementation contract for `score()` (determinism, non-mutation of inputs, tolerance for missing optional fields, thread-safety) and reaffirms the constructor-time-load discipline. The interface is deliberately method-agnostic: no mutual-information vocabulary in the ABC, so the Phase 2 `CalibratedScorer` (PRC-035) drops in without an interface break (PRC-010).
- `ScoreResult.timestamp_iso` field validator enforcing UTC-offset ISO 8601 strings: rejects naive datetimes and non-UTC offsets, normalises a trailing `Z` suffix to `+00:00` so behaviour is consistent across the `>=3.10` Python floor (Python 3.10's `datetime.fromisoformat` does not accept `Z`; 3.11+ does). Locks the timestamp convention down before PRC-013's `ViolationEvent` introduces a sibling field (PRC-010).

- `precept.scoring.embedding_proxy.EmbeddingProxy`: v0 concrete `Scorer`. Sentence-transformers cosine-similarity proxy on `all-MiniLM-L6-v2` by default (`paraphrase-MiniLM-L3-v2` and arbitrary HF model identifiers swappable via `model_name`). Per-field cosine on `contract.fields.required_fields`; unweighted-mean `overall_score`; missing target field scores `0.0` with `passed=False`; non-string field values stringified via `str()`; `method="embedding_cosine"` on every `FieldScore`; clamps cosine to `[0,1]` to absorb floating-point and adversarial-pair edges (PRC-011).
- Constructor-time model load with INFO log of the resident-memory footprint constant (no `psutil` dependency); `torch.set_grad_enabled(False)` set at construction for inference-only operation (PRC-011).
- `_skip_model_load: bool = False` test-only escape hatch on `EmbeddingProxy.__init__`: skips model load and causes `score()` to raise `RuntimeError` with explicit guidance. Used by unit tests to exercise scorer wiring without the ~80 MB `all-MiniLM-L6-v2` download. Documented as private; not part of the public API (PRC-011).
- Empty-`required_fields` contracts (ADR 0001 scaffold/observe-only mode) score `overall_score=1.0` with `field_scores=[]`. Vacuous pass aligns with PRC-013's `all(field_score.passed)` aggregation (PRC-011).
- Scorer-level `min_fidelity` fallback: when `contract.fields.min_fidelity` is `None` and `required_fields` is non-empty, `EmbeddingProxy` applies a `0.5` cosine threshold and logs a `WARNING` once per `score()` call. The default lives in the scorer, not in `ContractFields`, to keep the IR generic for the Phase 2 `CalibratedScorer` (PRC-035): a calibrated MI scorer would pick its own threshold from a calibrated probability distribution, where `0.5` cosine has no meaning. See `embedding_proxy` module docstring (PRC-011).

- `precept.evaluator` package scaffold (`Dev/src/precept/evaluator/__init__.py`) with empty `__all__`. Submodule names import directly until PRC-026 publishes the public surface (PRC-012).
- `precept.evaluator.rules` module exposing the three v0 rule evaluators as pure module-level functions: `required_fields_rule`, `preserved_entities_rule`, `forbidden_drops_rule`. Each is stateless, side-effect-free, and never raises - rule failures are returned as `RuleResult` data so the PRC-013 `Evaluator` aggregates them without exception-flow plumbing (PRC-012).
- `RuleResult` Pydantic model: `rule_name: str`, `passed: bool`, `details: dict[str, Any]` (default empty), `violation_message: str | None`. `ConfigDict(extra="forbid")`. `violation_message` is `None` on pass and human-readable on fail; per-rule `details` shape is documented in each rule's docstring and treated as a public contract for PRC-013's `ViolationEvent` renderer (PRC-012).
- `required_fields_rule(source, target, required)`: presence check on `target.fields`. `details = {"required": list[str], "missing": list[str]}` preserving the `required` order. `source` is part of the uniform rule signature but unused; documented in the docstring (PRC-012).
- `preserved_entities_rule(source, target, entities)`: case-insensitive substring matching on the newline-joined stringified values of `HandoffPayload.fields` (NOT `.raw`; `.raw` is optional and would couple rule behaviour to producer-population habits). Vacuous-pass when an entity is absent from source - mirrors `forbidden_drops_rule` semantics. NER-based matching deferred to PRC-030 (PRC-012).
- `forbidden_drops_rule(source, target, forbidden)`: structural drop check; flags fields present in `source.fields` but absent in `target.fields`. Target-only fields pass. `details = {"forbidden": list[str], "dropped": list[str]}` (PRC-012).

Hypothesis property tests: rules never raise on arbitrary input (random unicode keys, mixed-type values), and `required_fields_rule(any, target_with_those_fields)` always passes (PRC-012).

- `precept.types.ViolationEvent` Pydantic v2 model (`extra="forbid"`) carrying every field a downstream consumer needs to reconstruct an evaluation: `contract_name`, `contract_version`, `mode: Literal["block","warn"]`, `passed`, `score_result: ScoreResult`, `rule_results: list[RuleResult]`, `triggered_at_iso`, `event_id` (UUID4 default factory), `source_summary` / `target_summary: dict[str, str]`, and `schema_version: Literal["0.1"]`. The `triggered_at_iso` validator mirrors `ScoreResult._validate_timestamp_iso_utc` exactly (rejects naive and non-UTC offsets; normalises trailing `Z` to `+00:00` for the Python 3.10 `fromisoformat` floor) (PRC-013).
- `ViolationEvent.to_compact_dict() -> dict[str, str | int | float | bool]`: flat OTel-attribute layout under the `precept.*` namespace (no nested dicts at the top level) with a hard 4 KiB ceiling on the JSON-serialised byte length. Under size pressure, target-summary entries are dropped first (longest value first), then source-summary entries; a `precept.payload_truncated: True` flag is set on entry to the truncation loop, so it surfaces both the "we successfully truncated" and the irreducible-overflow ("we tried but the base shape exceeds the budget") cases. Setting the flag is idempotent so the ~36 flag bytes are paid exactly once across the whole loop. The 4 KiB target is the realistic OTel-attribute lower bound across backends (Datadog, Jaeger), so silently-dropped events are replaced with explicitly-flagged events that backends can detect and surface (PRC-013).
- `precept.errors.HandoffBlockedError(Exception)` with `violation_event: ViolationEvent` and `impact_summary: str = ""`. `impact_summary` is mutable by design: PRC-015's impact populator writes onto an existing instance immediately before re-raise, and PRC-014's integration layer is the only caller that constructs and re-raises this class. The mutability is a deliberate, documented contract that breaks the original PRC-014 ⇄ PRC-015 circular dependency without forcing the error class to depend on the impact-template module. A subprocess regression test asserts that `from precept.errors import HandoffBlockedError` does not transitively load `precept.evaluator.engine` or `precept.scoring.embedding_proxy` (PRC-013).
- `precept.exporters` package scaffold (`Dev/src/precept/exporters/__init__.py`) with empty `__all__`. Submodule names import directly until PRC-026 publishes the public surface, mirroring the `precept.evaluator` package convention (PRC-013).
- `precept.exporters.base.Exporter` ABC with abstract `export(event: ViolationEvent) -> None` and a `name: ClassVar[str]` requirement enforced by `__init_subclass__` (mirrors the `Scorer` pattern, including the `inspect.isabstract` exemption for intermediate abstract subclasses). Class docstring codifies the implementation contract: must not raise on transport failure (log + continue), must not mutate `event`, must be safe to call from multiple threads, and should be best-effort idempotent on repeat `event_id` (PRC-013).
- `precept.exporters.base.NoOpExporter` (`name="noop"`): default safe sink; `export()` is a no-op. Used by `Evaluator` when no exporter is supplied and as a baseline test fixture (PRC-013).
- `precept.exporters.base.MultiExporter` (`name="multi"`): fan-out exporter with per-child failure isolation. `export()` iterates children in declaration order; a `RuntimeError` from one child is logged and swallowed so siblings still receive the event. Empty child list is allowed (silent success) (PRC-013).
- `precept.evaluator.engine.Evaluator`: composes the three rule evaluators (PRC-012) with the injected `Scorer` (PRC-010) and `Exporter` (PRC-013) into a single `evaluate(source, target, contract) -> ViolationEvent` call. `passed` is the conjunction of every `RuleResult.passed` and every `FieldScore.passed`; `mode` is metadata. The evaluator never raises on a contract violation - the integration layer (PRC-014) decides whether to raise `HandoffBlockedError`. Exporter failures are isolated via try/except so an unconfigured or misbehaving exporter never starves the caller of its event. `source_summary` / `target_summary` are built from `payload.fields` with a 100-char-per-value upstream truncation (the 4 KiB compact-dict ceiling is the secondary bound) (PRC-013).
- Thread-safety contract on `Evaluator` (class docstring): one instance is safe to call from multiple threads, given the `Scorer` and `Exporter` contracts. Future caching or per-call mutation on `self` would break this and must update the contract loudly (PRC-013).

- `precept.integrations.langgraph.extractor.extract_payload(state, contract, *, max_field_chars=2000) -> HandoffPayload`: reads ONLY `contract.fields.required_fields ∪ preserved_entities` (de-duplicated, first-seen order) from arbitrary upstream state; no recursive descent, no PII heuristic, no stringification of non-string values. - `precept.contract.decorator.handoff_contract`: sync-only decorator frontend (ADR 0001's second frontend). Produces the same `HandoffContract` IR as the YAML loader by routing a nested `fields` mapping through `HandoffContract.__init__`, so schema violations surface as `ContractValidationError` at decoration time (not call time). Ergonomic kwarg `required` maps to IR `required_fields`. An optional first-positional sentinel detects the bare `@handoff_contract` misuse and raises `TypeError` with guidance rather than Python's native positional-argument error; omitting the required `name` raises `TypeError` (an API-usage error kept categorically distinct from a contract-content `ContractValidationError`); coroutine functions raise `TypeError` with the documented Phase-2 redirect. `functools.wraps` preserves the wrapped signature; the validated contract is attached as `__precept_contract__` by attribute assignment with a targeted `type: ignore[attr-defined]` (mypy-strict and ruff clean) for the PRC-014 integration layer to consume. No call-time evaluation at v0 (PRC-008).
